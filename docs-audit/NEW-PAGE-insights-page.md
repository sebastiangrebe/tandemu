# Suggested New Page: Insights Dashboard

**Proposed URL:** /docs/lead/insights

## Why This Page Is Needed

The `/insights` page exists in the frontend and is documented in CLAUDE.md, but has NO docs page. It's a significant feature showing ROI metrics that engineering leads care about deeply.

## Suggested Content

### Overview
The Insights page (`/insights`) provides AI investment analysis — helping engineering leads quantify the value of AI coding tools.

### Metrics

#### Productivity Multiplier
How much more output the team produces with AI assistance compared to manual-only coding. Calculated from AI ratio and throughput data.

#### Capacity Freed
Hours of developer time freed by AI assistance per week/month. Derived from AI-generated lines and estimated manual coding time.

#### Cost per Task
Average engineering cost per completed task. Combines session duration with team cost assumptions.

#### Throughput Chart
Tasks completed over time, showing trends in team velocity.

#### Cost Efficiency Chart
Cost per task over time — ideally trending downward as AI adoption increases.

#### Token Usage
AI token consumption patterns across the team.

#### AI Adoption Leaderboard
Per-developer ranking by AI code ratio, showing who's leveraging AI tools most effectively.

### How to Use Insights
- **For budget justification** — Show leadership the productivity multiplier and capacity freed
- **For team coaching** — Use the adoption leaderboard to identify developers who might benefit from AI coding training
- **For trend analysis** — Track cost per task over time to validate AI tool investment
